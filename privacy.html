Privacy Policy for Drive Link Exporter

Effective Date: October 2025

Drive Link Exporter (“the Extension”) is a Chrome extension designed to help users export Google Drive file metadata (such as file names, IDs, and links) from folders they select. We value your privacy and are committed to protecting your data. This policy explains what data we access, how we use it, and how it is protected.

1. Data Accessed

The extension requests read-only access to Google Drive metadata through the scope: https://www.googleapis.com/auth/drive.metadata.readonly. The Extension never reads, downloads, or modifies file contents. It only accesses metadata such as file name, ID, MIME type, size, and webViewLink/webContentLink values for the files within the Google Drive folder you explicitly choose.

2. Data Usage

The accessed metadata is used solely to generate a CSV file containing the file listing of the selected Google Drive folder. All processing occurs locally in your browser session. No data is transmitted to, stored on, or processed by any external servers.

3. Data Sharing

Drive Link Exporter does not share, sell, or disclose Google user data or metadata with any third party. The Extension does not use any analytics, advertising, or tracking services.

4. Data Storage & Protection

The Extension performs all operations in-memory on the client side. No Google user data is stored persistently by the Extension. OAuth tokens are securely handled by Google and Chrome according to standard Google authentication practices.

5. Security Measures for Google User Data

We implement the following data protection mechanisms for Google user data accessed via the drive.metadata.readonly scope:

  • Local-only processing: Google Drive metadata is processed locally within the browser session. The Extension does not operate any backend servers or external processing services.
  • No external storage: The Extension does not store Google Drive metadata on external servers or in persistent storage. Data is held in-memory only for the purpose of generating the CSV.
  • Secure authentication (OAuth 2.0): Access is granted only after the user completes Google’s OAuth 2.0 consent flow. Tokens are issued and managed using Google and Chrome’s standard authentication mechanisms.
  • Encrypted transport: Communication between the Extension (via the user’s browser) and Google APIs is protected by HTTPS/TLS encryption.
  • No advertising, profiling, or AI/ML training: Google user data is not used for advertising, analytics profiling, marketing, or training machine learning models.

6. Data Retention & Deletion

The Extension does not store or retain any user data. Once the session ends or the browser popup is closed, all accessed data is cleared. Users may also revoke the Extension’s access anytime from their Google Account permissions page (https://myaccount.google.com/permissions).

7. Compliance with Google API Services User Data Policy

This Extension fully complies with the Google API Services User Data Policy , including the Limited Use requirements. Google user data is used solely to provide the Extension’s core functionality (exporting Google Drive file metadata selected by the user into a CSV).

8. Contact

If you have questions about this policy or how the Extension handles your data, please contact us at: admin@rohitsprojects.store
